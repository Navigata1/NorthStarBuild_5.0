### 14.4.1 AUTOMATION SEQUENCING PRINCIPLE

> "Don't use RALPH until you've done the manual reps."

```
AUTOMATION SEQUENCING — Earn Automation Through Understanding
─────────────────────────────────────────────────────────────────────────────

RATIONALE:
• Manual execution builds understanding
• Automation without understanding = brittle systems
• Debugging requires knowing what "normal" looks like
• Premature automation hides root causes

SEQUENCING PROTOCOL:

1. MANUAL FIRST (3-5 reps)
   □ Execute the process manually
   □ Note variations and edge cases
   □ Document decision points
   □ Identify what can go wrong

2. DOCUMENT
   □ Write down the steps
   □ Include "why" not just "what"
   □ Capture success criteria

3. SEMI-AUTOMATE
   □ Script the repetitive parts
   □ Keep decision points manual
   □ Verify outputs match manual runs

4. FULL AUTOMATE (Only when confidence is HIGH)
   □ Automate remaining steps
   □ Add monitoring and alerts
   □ Include rollback capability
   □ Test failure scenarios

ANTI-PATTERNS:
✗ Automating before understanding
✗ "It works, ship it" without comprehension
✗ Delegating without capability to verify
✗ Skipping the manual learning phase
```
